# Pixelle-Video Project Notes

## Documentation Discipline

- Any code change that affects product behavior, routes, UX, API contracts, resource layout, or developer workflow must update the relevant documentation in the same turn.
- Prefer updating the highest-signal docs first: `README.md`, `README_EN.md`, `frontend/README.md`, and the matching files under `docs/zh/` and `docs/en/`.
- When a feature introduces new runtime resources or directories, document the canonical locations explicitly.

## This Project Preference

- The user expects documentation to stay in sync with implementation by default.
- Do not treat doc updates as optional follow-up work for this repository.
