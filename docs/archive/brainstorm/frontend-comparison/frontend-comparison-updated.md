# Pixelle-Video 双前端综合对比分析报告（更新版）

> 分析时间：2026-04-21（更新后）
> 旧前端：`web/` (Streamlit) — **已删除**
> 新前端：`frontend/` (Next.js 16 + React 19) — **默认入口**
>
> 2026-04-22 二次更新：已完成 i18n、模板静态/实时预览、Quick 媒体测试预览、Preset CRUD、Workflow 编辑、设置页运行时能力、历史 storyboard 详情与终态删除；下文缺口状态仅代表本报告撰写时的快照。

---

## 重大变化摘要

自上次审查以来，项目发生了**结构性转变**：

1. **旧前端 Streamlit 已被删除** — 所有 Python 源文件从 `web/` 移除，仅剩 `__pycache__` 缓存
2. **docker-compose 默认切换为 Next.js** — `frontend` 服务成为默认，`streamlit` 降为 `profiles: ["legacy"]` 且默认只读
3. **新前端功能暴增** — 从 75-80% 完成度跃升至 **90-95%**
4. **GitHub Actions 前端工作流上线** — `.github/workflows/frontend.yml`

## 六、用户体验对比

### 6.1 具体场景（更新后）

**场景：创建批量任务**
- **旧前端**：文本框输入多个主题，换行分隔，100 条上限
- **新前端**：**支持 CSV 导入** + 手动行编辑 + Pipeline 选择器 + 批量名称自动生成

**场景：查看素材库**
- **旧前端**：无素材库概念，只有历史记录
- **新前端**：5 个分类（Videos / Images / Voices / BGM / Scripts），网格/表格视图，筛选栏

**场景：系统设置**
- **旧前端**：长表单，LLM + ComfyUI 挤在一起
- **新前端**：**Tabs 分组**（API Keys / Appearance / Storage / About），更清晰

**场景：管理 Workflows**
- **旧前端**：下拉选择，无管理界面
- **新前端**：独立页面，Self-host / RunningHub 分类，详情查看

### 6.2 新前端独有体验

| 体验 | 说明 |
|------|------|
| **Skeleton 加载** | 列表加载时有骨架屏，不闪白 |
| **Toast 通知** | 操作成功/失败有全局 toast，不阻塞操作 |
| **Error Boundary** | 页面局部崩溃不会导致整个应用白屏 |
| **暗色模式** | next-themes 一键切换 |
| **Sidebar 折叠** | 可收起，localStorage 持久化 |
| **URL 恢复任务** | 刷新后从 URL `task_id` 恢复进度 |
| **项目强制校验** | 未选项目时 Dialog 拦截，不生成 |

## 附录：docker-compose 服务变更

```yaml
# 更新前（旧前端为主）
services:
  api:     port 8000   # FastAPI
  web:     port 8501   # Streamlit（默认）

# 更新后（新前端为主）
services:
  backend:   port 8000   # FastAPI（改名）
  frontend:  port 3000   # Next.js（默认启动）
  streamlit: port 8501   # Streamlit（profiles: ["legacy"], 只读）
```

启动命令变化：
```bash
# 默认启动（新前端 + 后端）
docker-compose up -d

# 如需旧前端（显式启用 legacy）
docker-compose --profile legacy up -d
```
